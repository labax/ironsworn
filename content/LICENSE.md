# Content licensing

Application source code and bundled content are licensed, reviewed, and tracked separately.

- Source code is covered by the repository-level `LICENSE` file.
- Content in this directory must not be assumed to share the source-code license.
- Approved SRD-derived data, original content, examples, manifests, attributions, and third-party notices must be documented before release.
- Do not add unapproved Ironsworn rulebook prose, official art, icons, layout, trade dress, or other protected material.

Future content imports should identify source, license, transformation notes, attribution requirements, and review status in the content manifest or related licensing documentation.
