# Oracle content licensing review checklist

Date prepared: 2026-07-13  
Reviewer field default: `OR-12 reviewer` until a named release reviewer signs a row.  
Scope: bundled oracle tables in `src/app/domain/oracles/oracle-browser.ts`, provenance records in `content/manifest.json`, runtime custom oracle storage, public-build content gating, and OR-11 oracle UAT evidence.

This checklist is a repeatable release artifact, not legal advice. Do not mark a table release-eligible unless its source, license/status, attribution or notice needs, provenance record, reviewer, date, and evidence are documented. Missing, conflicting, unknown, unreviewed, or blocked provenance is a release blocker and must not be silently accepted.

## Required release decision states

| State   | Meaning                                                                                                                    | Public build handling                                                                       |
| ------- | -------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| Pass    | Evidence confirms project-original or otherwise documented rights, `releaseStatus: allowed`, and `reviewStatus: reviewed`. | May be included only through release-eligible provenance gates.                             |
| Fail    | Evidence disproves eligibility or shows prohibited protected material.                                                     | Must be removed or changed before release.                                                  |
| Blocker | Evidence is missing, conflicting, unknown, unreviewed, review-required, or blocked.                                        | Must be excluded from public builds.                                                        |
| Skipped | Check could not run; limitation is recorded precisely.                                                                     | Treat affected release decision as blocker unless another listed evidence source covers it. |

## Future table addition checklist

For each new bundled oracle table, add or verify all fields before release:

- [ ] Stable oracle table ID and stable manifest ID are recorded.
- [ ] Manifest entry source, license/status, attribution or notice reference, provenance reference, reviewer, review date, and review state are complete.
- [ ] `releaseStatus` is `review-required` or `blocked` until all evidence is complete; never use `allowed` for unknown or unreviewed records.
- [ ] Any official, third-party, SRD-derived, or copied material has documented compatible rights and required notices before inclusion.
- [ ] Official art, icons, copied layouts, screenshots, trade dress, and unapproved official prose are absent.
- [ ] `npm run validate:content`, `npm run validate:content-json`, and `npm run test:ci` pass.
- [ ] Browser/resolver/result/context/history/journal/search/custom/responsive/content-gating evidence is updated.
- [ ] Skipped manual checks and environment limitations are recorded precisely.
- [ ] Any missing or conflicting provenance is listed as an unresolved release blocker.
