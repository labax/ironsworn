# Onboarding bundled-copy review

Scope: bundled onboarding-facing strings in welcome, character setup, first-vow setup, linked progress review, final review/completion, skip/exit, validation/error paths, responsive states, tests, package scripts, and provenance records.

This is a release artifact, not legal advice. Do not mark copy release-eligible if it copies official prose, teaches rules in tutorial form, promises unsupported automation, or mixes user-authored campaign content with bundled UI text.

## Terminology glossary

| Term                 | Use                                                                                                           |
| -------------------- | ------------------------------------------------------------------------------------------------------------- |
| Unofficial companion | Required disclaimer term for the app relationship to Ironsworn rightsholders.                                 |
| Campaign workspace   | Local browser workspace that stores user campaign records. Do not imply account sync or cloud backup.         |
| Character            | User-created character record.                                                                                |
| Core stats           | Edge, Heart, Iron, Shadow, and Wits labels.                                                                   |
| Status tracks        | Health, Spirit, and Supply.                                                                                   |
| Momentum             | Current Momentum, reset, and maximum values.                                                                  |
| Vow                  | User-authored promise record. Do not provide official examples.                                               |
| Progress track       | Track linked to a vow or created elsewhere. Use “linked progress track” for the first-vow connection.         |
| Skip for now         | Leaves setup incomplete and opens the workspace.                                                              |
| Exit setup           | Leaves setup and opens the workspace; saved records remain, unsaved text may be discarded after confirmation. |
| Back                 | Returns to the previous setup step and preserves the current first-vow draft.                                 |
| Continue             | Moves to the next setup step after review.                                                                    |
| Review setup         | Creates the first-vow linked track and shows the local review panel.                                          |
| Complete setup       | Validates required setup records, saves completion status, and opens the workspace.                           |

## Contributor checklist for future onboarding-copy changes

- Keep bundled UI text project-original; do not copy official prose, examples, art descriptions, screenshots, icons, layouts, or trade dress.
- Keep onboarding short; do not turn setup into a rules tutorial.
- Check behavior truthfulness for local browser storage, save failures, skip, exit, Back, review, and completion.
- Preserve the unofficial-product disclaimer where users first enter onboarding.
- Keep user-authored character, vow, journal, oracle, and notes text out of bundled content approval.
- Update this review, `content/manifest.json`, and tests when onboarding-facing strings change.
- Mark questionable copy as blocked or review-required instead of silently approving it.

## OB-10 manual review evidence

- Desktop width review: checked source-rendered responsive wrappers for welcome, first-vow, and final review cards; long messages use normal paragraphs, helper text, lists, or definition lists that can wrap.
- Mobile width review: checked onboarding CSS breakpoints and form action wrapping for narrow screens; action groups wrap or stack and long helper/error messages are not fixed-width.
- Blockers: none recorded for current bundled onboarding text.
