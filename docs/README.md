# Documentation

This folder collects project requirements, architecture notes, licensing guidance, release notes, testing guidance, product planning, and decision records.

## Areas

- [Architecture](architecture/README.md): technical direction and system structure.
- [Decisions](decisions/README.md): architecture and product decision records.
- [Licensing](licensing/README.md): source/content licensing guidance and review notes.
- [Product](product/README.md): MVP goals, requirements, and user-facing scope.
- [Release](release/README.md): release notes and release process placeholders.
- [Testing](testing/README.md): testing strategy and locations.

Existing and future requirements documents should be linked from the relevant area above rather than scattered across the repository root.
