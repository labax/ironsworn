# Product notes

Use this folder for MVP scope, release addenda, user stories, requirements, and product planning notes.

## Document templates

Reusable Markdown templates are available in [`templates/`](templates/README.md). The template set covers:

- Business and product requirements.
- MVP scope and version-specific release addenda.
- Functional and rules-engine requirements.
- Data / domain modeling.
- UX flows and wireframe requirements.
- Content and licensing requirements.
- Non-functional requirements.
- Acceptance criteria and test planning.

Start with the template index, choose only the documents needed for the release, replace every placeholder, and keep requirement IDs stable after review begins.

## Documentation expectations

- Link completed documents from this product index or a release-specific index rather than scattering them across the repository.
- Keep business outcomes, observable functional behavior, deterministic rules, durable data, UX, quality attributes, content licensing, and tests traceable but separate.
- Record unresolved decisions explicitly.
- Do not add unapproved official Ironsworn prose, move text, oracle text, asset text, art, icons, screenshots, copied layout, or trade dress.
